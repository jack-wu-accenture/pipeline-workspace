## Summary

The Data Management Capability Assessment Model (DCAM® v2.2.2) is the EDM Council framework that Novartis uses as its global data governance standard. Any China MDM replacement solution must demonstrate alignment with DCAM principles.

## DCAM Framework Scope

DCAM defines capabilities across the full spectrum of data management:
- Data Governance (Framework, Policies, People, Accountability)
- Data Integration (ETL, APIs, Real-time Sync)
- Data Quality (Validation, Cleansing, Monitoring)
- Data Modeling (Schemas, Metadata, Hierarchy)
- Data Lifecycle Management (Collect → Create → Update → Archive → Distribute)
- Data Security & Compliance (Access Control, Audit Trails, PIPL/GDPR/HIPAA)

## Novartis 2026 DCAM Expectations

A separate Excel file (`DCAM 2026 Expectations.xlsx`) defines Novartis-specific DCAM targets for 2026 across 5 worksheets. This represents the governance maturity bar the replacement solution must meet or exceed.

## Implication for This Engagement

- Workstream B (EDM/Governance Impact Assessment) must assess the proposed solution against the DCAM 2026 Expectations
- The AWS Custom solution architecture must embed DCAM-aligned governance capabilities by design
- Any local vendor (e.g., Sunway) must demonstrate DCAM capability coverage
