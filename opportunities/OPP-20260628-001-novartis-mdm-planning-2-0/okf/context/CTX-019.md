## MDM 选型评估标准与权重（已内部达成共识）

| 维度 | 权重 | 关键考察点 |
|---|---|---|
| **Function Fit Gap（功能契合度）** | **30%** | DCR全覆盖(Create/Update/Merge for HCO/HCP)、批量处理、Match & Merge灵活性、多级关系管理、审计能力 |
| **Flexibility（灵活性）** | **20%** | 数据模型可扩展、API/事件/批量集成、数据导入导出工具、报告分析、CI/CD支持 |
| **Security & Compliance（安全合规）** | **20%** | PIPL/CSL合规、数据在华存储、细粒度权限+审计、加密、安全事件响应 |
| **Supportability & Sustainability（可支持性）** | **15%** | 本地化支持SLA、供应商生命力、升级维护便利性、环境可持续性 |
| **Cost & Timeline（成本周期）** | **15%** | 软件+运维TCO、实施周期与资源、未来升级扩展成本 |

## MDM 数据模型（当前 EBX 基础）

**核心实体：**
- **HCP**（医生）: MDM ID, Status, Remote Key, 专科、证书、发表文章、照片
- **HCO**（医疗机构）: 医院/药店/经销商，总分院/总部门店层级
- **Department**（科室）: 挂载在 HCO 下
- **Affiliation**（关联关系）: HCP ↔ HCO, HCO ↔ HCO 层级
- **Address**（地址）
- **Geo Unit**（地理单元）: 3级 — 省份/城市/区县
- **Reference Data**（参考数据）: LOV + Remote Key 映射
- **DCR Header/Line**（变更请求记录）

## 集成架构

**数据摄入（Ingestion）：**
- 事件驱动: Salesforce CRM → Solace → MuleSoft → SCDF
- 文件批量: Novartis Connect/SWIFT/SharePoint → AXway API MFT → Solace → SCDF

**数据发布（Publish）：**
- 事件驱动: → Solace → MuleSoft → Salesforce CRM
- 弹性分发: → Solace → Elastic → SWIFT/Novartis Connect/下游系统

## 性能要求（关键约束）

- HCO 批量 DCR：每个 SWIFT 文件 **5,000–6,000 条记录**，每月固定日期推送，要求在规定时间窗口内完成处理
- Match & Merge：SCDF + Elasticsearch 批量去重；EBX Match Plugin 非批量评分
- 批量审核：支持部分提交（Partial Batch Submission），已验证记录立即处理，无需等待整批完成

## DCR Merge 业务逻辑（HCP）

- 转院场景 vs 非转院场景逻辑不同
- Winner/Loser 机制：可 SWAP、可选 Winning Attributes
- Merge 结果：Loser 失活，HCP 属性按 winning attributes 更新，Affiliation 合并（逻辑因转院类型不同）
