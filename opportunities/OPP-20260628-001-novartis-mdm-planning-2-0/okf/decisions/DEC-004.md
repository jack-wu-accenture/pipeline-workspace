## Decision

**推荐方案：AWS China 自建 MDM 核心引擎，外挂层保持不动**

替换范围仅限 MDM 核心（实体管理、Match & Merge、数据模型）。
现有 Spring Boot/Cloud 外挂逻辑不重写，仅替换调用的 API 从 EBX → 新引擎。

## 决策依据

1. 客户已有自己的判断：只替换核心模块，控制技术复杂度（非我们说服，是客户驱动）
2. 2025 Reltio PoC 已验证 MDM 方向可行，AWS 路线是新 PoC 验证重点
3. 外挂层重写会使工作量和风险倍增，超出 $1M USD 实施预算
4. Pimcore 架构参考验证了"配置驱动实体模型 + 关系型DB + 独立 matching"模式的可行性

## 对比选项

| | Pimcore 开源方案| AWS 自建（推荐）|
|-|--------------------|-----------------|
| 许可证成本 | 零（开源）但定制成本高 | 零（全开源）|
| 外挂兼容 | PHP/Java 生态不兼容 | 最小化改动 |
| 平台锁定 | 高 | 低 |
| 历史先例 | 无中国实施案例 | 2025 PoC 基础 |

## Decision Owner

Novartis China IT + Global EDM（Go/No-go 由双方共同决策，Nov-Dec 2026）

## 验证方式

WS-E 最小化 PoC → Go/No-go 就绪报告（2026 M5）
