## Requirement

The replacement MDM platform must preserve and enhance all 2025 BRS-defined functional requirements. These are non-negotiable — they represent existing business processes already in production on EBX.

## Critical DCR Processes (Must Preserve)

**HCP Data Change Requests:**
- DCR-Create from New CRM (offline batch review + single review)
- DCR-Update from New CRM
- DCR-Merge from New CRM
- DCR-Create from Customer Connect / DS Batch Upload (offline + online batch review)
- DCR-Update and Merge from Customer Connect / DS Batch Upload

**HCO Data Change Requests:**
- DCR-Create, Update, Merge from SWIFT

## Core Functional Areas (All 7 Must Be Covered)

- FR-1: Master Data Model Management (entity, attribute, relationship, RDM, GEO)
- FR-2: Data Lifecycle (ingestion, consumption, DCR maintenance, alias)
- FR-3: Data Quality (rules, execution, dashboard)
- FR-4: Data Services (sync/distribution, reporting, monitoring)
- FR-5: Process Management (workflow engine)
- FR-6: Security (access control, operational permissions)
- FR-7: System Admin (audit, user management)

## Integration Must Support

- Inbound: New CRM, Customer Connect, DS Batch Upload, SWIFT
- Outbound: All existing downstream consuming systems
- Non-functional: Performance metrics as per BAU requirements in BRS

## Constraint

This is a replacement, not a redesign. Functional parity with existing EBX behavior is the baseline — enhancements on top of parity are welcome but cannot come at the cost of baseline coverage.
