## Phase 1 — 需求对焦

### 背景修正（来自用户输入）

**RFP核心驱动力：**
EBX 许可证约 2 年后到期，Global 无法负担资金，China 必须在有限预算内找到保证业务可持续性的替代路径。核心命题是：**用有限的钱，在 EBX 到期前完成迁移并保持业务不中断。**

**需求修正：**
- REQ-005/REQ-006（系统复杂度证据化）→ **已过时/降权**。客户已做过 Reltio PoC，形成了"只替换核心模块可控制复杂度"的自有判断，靠复杂度说服客户行不通。
- REQ-004（并行两条路线）→ **客户已倾向自开发路线**，Reltio 因 Global 资金问题排除，Local Vendor 待评估。
- REQ-001 → **重新定义**：核心任务不是"建立评估标准"，而是**验证自开发 MDM 方案技术可行性**（核心模块替换策略是否成立）。

**两个核心需求（用户确认）：**

**核心 REQ-A（原 REQ-001/012 合并）：**
验证自开发 MDM 方案（替换核心模块）的技术可行性——
- 哪些 EBX 能力必须自建？哪些可借助 AWS 原生能力？
- 核心模块替换边界如何划定？
- PoC 验证范围最小化到什么程度可支撑 Go/No-go 决策？

**核心 REQ-B（新增）：**
在有限预算 + 长周期分批付款（2026 H2 → 2028 年底）下，如何安排项目计划——
- 如何分阶段控制开发投入，避免前期大量消耗预算？
- 长周期下如何控制沟通、管理、变更成本？
- 付款节奏与交付里程碑如何匹配，保持客户信心？

### 需求优先级（初稿）
